# clients/

Internal scoping notes for retained client engagements. Not linked from the public
landing page. Each subdirectory is one engagement.

## Conventions

- Each client gets a directory: `clients/<slug>/`.
- Inside, at minimum: a `README.md` describing the engagement scope, status,
  cadence, and any non-public references.
- Anything client-sensitive (their ticket data, their internal docs, their
  product names if not already public) does not live here. It lives in the
  client's own systems and is referenced by link, not embedded.
- Public-facing writing about an engagement lives in `writing/`, not here.
